การตีพิมพ์หรือออกจำหน่าย (Publish) สำหรับระบบ Android

โดย นิป เอมรัฐ
๒๐ ก.พ. ๒๕๖๒

การจำหน่ายขึ้น Google Play Store สำหรับอุปกรณ์ระบบ Android

ต้องมีบัญชี Google Play Developer account (ค่าสมัคร $25 จ่ายครั้งเดียว)

สมัครได้ที่นี่ https://play.google.com/apps/publish/signup/

Design assets

  • ไอคอน Icon ขนาด (512 x 512 px)
  • ภาพหน้าจอของแอป App screenshots
  • ภาพประกอบขนาด 1024 x 500 px)

คุณภาพของแอปเป็นสิ่งสำคัญ แนะนำให้อ่านคำแนะนำของ Google ที่นี่ https://play.google.com/about/developer-content-policy/#!?modal_active=none

การเตรียมการ

  1. จะต้องระบุ app icon, name, package name และ version ของแอปก่อน
    ดูคำแนะนำที่นี่ https://docs.thunkable.com/thunkable-cross-platform/create/app-icon-+-name
  2. ชื่อของแอปที่ package name จะต้องไม่ซ้ำกับแอปอื่นที่อยู่บน Play Store
  3. นอกจากนี้ ยังต้องระบุ Build และ Version Number
  4. เลือก Download หรือ Publish > Publish to Google Play Store (Android)
    จะได้ไฟล์ .apk ดาวน์โหลดลงมา

ขั้นตอนการส่งเพื่อตรวจสอบ

  1. ที่ Console ของ Google Play Developer สร้างแอปใหม่โดยคลิกปุ่ม Create Application
  2. เลือกคำสั่ง App releases
  3. คลิกปุ่ม Create Releases
  4. เปิดการใช้ Google Play App signing
  5. ลากไฟล์แอป (.apk) ไปวางยัง console
  6. พิมพ์ questionnaire ระบุราคา pricing และ availability
  7. ใส่ภาพประกอบอย่างน้อย 4 design assets - 2 screenshots, ไอคอนความละเอียดสูง (512 x 512 px) และภาพกราฟิกประกอบ (1024 x 500 px)

สร้างแบบทดสอบ (Quiz)

โดย นิป เอมรัฐ
๙ ก.พ. ๒๕๖๒

แอปทดสอบความรู้ น่าจะเป็นสิ่งที่ครูทุกคนปรารถนาที่จะสร้างด้วยมือของตัวเอง เรามาลองทำกัน

  1. จาก User Interface ลาก Label 2 อันไปวางยังโทรศัพท์
    เราจะใช้ Label1 แสดงความก้าวหน้าและ Label2 แสดงคำถาม
    พิมพ์ "จงตอบคำถามต่อไปนี้" ลงใน Label1 และพิมพ์ "ธงชาติไทยมีกี่สี?" ลงใน Label2
  2. ลาก “Text Input” อันไปวางยังด้านล่างของโทรศัพท์ เราจะใช้มันเพื่อให้ผู้ใช้พิมพ์คำตอบลงไป
  3. ลาก Button อันไปวางยังโทรศัพท์ ตั้งชื่อว่า "ส่งคำตอบ"
  4. เปิดลิ้นชัก Button1 ลาก “when Button1 Click” ออกมาวางยัง blocks editor จะมีเสียงดังขึ้น เมื่อวางถูกตำแหน่ง
  5. เปิดลิ้นชัก “Control” ลาก “if” มาวางไว้ที่ภายในของ “when Button1 Click”
  6. จากนั้น เปิดลิ้นชัก "Logic" ลากบล็อก equal มาวางไว้ที่ภายในซ็อกเก็ต "if"
    หากค่าภายในซ็อกเก็ต "if" เป็นจริง มันจะสั่งให้บล็อกที่อยู่ใน "do" ทำงาน
    แต่หากไม่เป็นจริง บล็อก "do" ก็จะไม่ทำงาน
  7. เปิดลิ้นชัก TextInput1 ลาก “from TextInput1 get Text” ออกมาวางไว้ภายในบล็อก equals เครื่องหมายเท่ากับ
  8. เปิดลิ้นชัก text ลากบล็อก text มาวางไว้ที่ภายในบล็อก equals ด้านขวา
    พิมพ์คำตอบที่ถูกต้องลงในบล็อก text (เช่น 3 สี)
    บล็อก equals นี้จะตรวจดูว่าคำตอบที่ผู้ใช้ตอบนั้น ถูกต้องหรือไม่
    พึงระลึกว่า คำตอบที่พิมพ์เข้ามานั้น จะเป็น case and space sensitive หมายความว่า ตัวอักษรภาษาอังกฤษตัวพิมพ์ใหญ่และเล็กนั้นถือว่าเป็นคนละตัวกัน และการเว้นวรรคก็ต้องเว้นให้ตรงกับคำเฉลยด้วย
  9. หากข้อความที่พิมพ์ตอบเข้ามาตรงกับคำตอบที่ถูกต้อง (3 สี) label แรกจะถูกเปลี่ยนเป็นข้อความว่า "ถูกต้อง!" แล้ว label ที่สองจะแสดงคำถามที่สองต่อไป
    สมมติว่า คำถามที่สอง คือ ธงชาติไทยมี 3 สี ได้แก่ สีน้ำเงิน สีแดง และสีอะไร?
  10. ให้ใช้บล็อก “from label1 set text” ภายในลิ้นชัก Label1 กำหนดข้อความว่า "ถูกต้อง!" ส่วนของ Label2 ก็ให้ทำเช่นเดียวกัน โดยแสดงเป็นคำถามที่สอง
  11. คลิก Live Test และทดสอบแอปในเครื่องของคุณว่าทำถูกต้องดังที่หวังไว้หรือไม่
  12. ทำข้อต่อๆ ไป โดยคลิกขวาที่บล็อก if เลือก duplicate ซึ่งจะสำเนาบล็อก if สำหรับใช้กับคำถามข้อ 2
  13. ใช้วิธีการเดียวกันนี้ ในการทำคำถามข้อที่เหลือทั้งหมด
    ในคำถามข้อสุดท้าย ให้แสดงข้อความกล่าวถึงการเสร็จสิ้นการทดสอบ เช่น "ยินดีด้วย คุณตอบคำถามทั้งหมดเกี่ยวกับธงชาติไทยได้อย่างถูกต้อง"
  14. คลิก Live Test เพื่อทดสอบอีกครั้ง
  15. ไชโย!

ภาพเคลื่อนไหว

โดย นิป เอมรัฐ
๗ ก.พ. ๒๕๖๒

ภาพเคลื่อนไหว (animation) ช่วยดึงดูดความสนใจได้ดี และยังดูน่ารักด้วย มาดูวิธีการใส่ภาพลงในแอปกัน

  1. ก่อนอื่น เตรียมรูปภาพที่ต้องการ โดยอาจจะสร้างขึ้นเองจากโปรแกรมต่างๆ เช่น Adobe After Effects หรือจะหาจาก lottiefiles.com* ซึ่งเป็นเว็บไซต์แชร์ภาพเคลื่อนไหวก็ได้
  2. จาก Image ลาก "Animation" ไปวางยังโทรศัพท์
  3. อัพโหลดไฟล์ภาพขึ้นไป โดยคลิกที่ช่อง Animation ในแถบ Animation1 ทางขวามือ คลิก Upload files เลือกไฟล์ภาพที่ต้องการ (*.json) แล้วคลิกปุ่ม Open
  4. ปรับแต่งค่าขนาดความสูงและความกว้างของภาพตามต้องการ** เช่น หากต้องการให้ภาพแสดงเต็มจอ ก็ให้ปรับ Height และ Width ให้ช่องบนเป็น Pick Ones: Fit contents, Fill container และช่องล่างเป็น Fill container

*เพื่อความสะดวก อาจติดตั้งแอป Lottie Preview เพื่อดูภาพเคลื่อนไหวในอุปกรณ์ของคุณ ก่อนจะนำมาใช้จริง (preview)

** เลือก Relative Size (e.g. "50%"") หากต้องการระบุขนาดภาพให้สัมพันธ์กับขนาดจริงเป็นเปอร์เซ็นต์ และเลือก Absolute Size เมื่อต้องการระบุขนาดของภาพให้ตรงตามตัวเลขที่พิมพ์ลงไป

เครื่องแปลภาษาสามารถแปลได้หลายภาษาตามที่เรากำหนด ลองแปลภาษาไทยเป็นภาษาอังกฤษกันดู

  1. จาก User Interface ลาก "Button" ไปวางยังโทรศัพท์
  2. จาก User Interface ลาก "TextInput" ไปวางยังโทรศัพท์
  3. เราจะปรับขนาดของ TextInput โดยที่คุณลักษณะด้านขวา เลือก height เป็น "Relative Size" และพิมพ์ 20% ลงในช่องด้านล่าง
    ขณะเดียวกัน ก็เลือก width เป็น "Relative Size" และพิมพ์ 80% ลงในช่องด้านล่างเช่นกัน
  4. ที่ Voice ลาก "Text-to-Speech" ไปวางยังโทรศัพท์
  5. ที่ Voice ลาก "Translator" ไปวางยังโทรศัพท์
  6. ที่คุณลักษณะของ Translator ซึ่งอยู่ด้านขวามือ เราจะเปลี่ยน/เลือกภาษาที่นี่ ภาษาต้นทาง source language คือภาษาที่ผู้ใช้ถนัด ส่วนภาษาปลายทาง target lanugage จะเป็นภาษาที่เราต้องการให้แปลไปเป็น ในตัวอย่างนี้ เราจะเลือกภาษาไทยเป็นภาษาต้นทาง เพื่อเปลี่ยนหรือแปลไปเป็นภาษาปลายทาง คือ ภาษาอังกฤษ
  7. คลิกแท็บ Blocks เพื่อเริ่มเขียนโปรแกรม
  8. คลิก Button1 เพื่อเปิดลิ้นชักของ Button1 ออกมา
  9. ลาก “when Button1 Click” ออกมาวางยัง blocks editor
  10. เปิดลิ้นชัก Translator1 ลาก "in Translator1 call Translate" มาวางที่ "when Button1 Click" จะมีเสียงดังขึ้น เมื่อวางถูกตำแหน่ง
  11. เปิดลิ้นชัก TextInput1 ลาก "from TextInput1 get Text" ออกมาวางยัง blocks editor เชื่อมต่อกับซ็อกเก็ต "text to translate."
  12. ที่ลิ้นชัก Text_to_Speech1 ลาก "in Text-to-Speech1 call Speak" มาวางที่ Translator1
  13. ลากบล็อกผลลัพธ์จาก "in Translator1 call Translate" มาวางที่ซ็อกเก็ต "text" ของบล็อก Text-to-Speech
  14. คลิก Live Test
  15. พิมพ์ข้อความหรือประโยคภาษาไทยลงไป แตะคำว่า Button ในแอป โทรศัพท์ของคุณจะพูดแปลออกมาเป็นภาษาอังกฤษ
    ยินดีด้วย คุณเพิ่งจะสร้างแอปแปลภาษาได้สำเร็จ ไช..โย!

Thunkable

ใครๆ ก็สามารถสร้างแอปของตัวเองได้ง่ายๆ ด้วย Thunkable

แทนที่จะต้องเขียนรหัสคอมพิวเตอร์ (code) เราเพียงลากส่วนประกอบ (components) นำมาวางเชื่อมต่อกันเป็นบล็อก (blocks) งานที่สร้างขึ้นสามารถใช้ได้ทั้งในอุปกรณ์ที่ใช้ระบบ Android และ iOS (cross platform) นอกจากนี้ เรายังสามารถนำตัวอย่างงานจากห้องแสดงผลงาน (gallery) มาปรับแต่งใช้งานได้ หากมีปัญหาติดขัดหรือคำถามใดๆ ก็ยังมีชุมชนผู้ใช้จำนวนมากที่คอยช่วยเหลือ ที่สำคัญคือ เราสามารถใช้ได้ฟรี โดยไม่จำกัดจำนวนงาน

Thunkable มีสองรุ่น คือ Thunkable Classic Android สำหรับอุปกรณ์ระบบ Android และ Thunkable X หรือ Thunkable Cross Platform ที่สามารถใช้ได้ทั้งระบบ Android และระบบ iOS

บรืษัท Thunkable (thunkable.com) ก่อตั้งโดย Arun Saigal และ WeiHua James Li
